# level-interview

<details>
<summary><h2>📌 스터디 기본 룰</h2></summary>

### 1. 진행 방법
- 질문 도출
- 질문 답변
- 회고

### 2. 문서 작성 기준
- **질문 도출**: 공유 문서에서 진행
- **질문 답변**: 공유 문서에서 **접는 글(Details)** 형태로 작성

### 3. 질문 도출 기준
- 레벨에서 애매했던 개념이나 새롭게 배운 개념
- 미션 구현 중 “왜 이렇게 했지?” 싶었던 부분
- 참고: [우테코 레벨 인터뷰](https://github.com/woowacourse-study-6th/level-interview) 문항 활용 가능

### 4. 질문 답변 템플릿
> 💡 **핵심**: 솔직하게 '틀린 생각'을 남겨두는 것도 중요합니다. 그래야 나중에 성장을 확인할 수 있습니다.

<details>
<summary><b>질문 : (여기에 질문을 작성하세요)</b></summary>
<div markdown="1">

<br>

**내 생각 :**
- (내가 알고 있는 개념 기반 설명)
- (왜 그렇게 생각하는지 근거)

**의문점 :**
- (헷갈리는 부분 / 확신 없는 부분)

**배움 후 수정된 생각 :**
- (다른 의견 반영)

**결론 :**
- (내 기준 정리, 언제 어떻게 쓸지)

</div>
</details>

### 5. 스터디 발표
- 한 명씩 돌아가면서 준비한 질문 내에서 **질문과 꼬리 질문**을 주고받으며 진행
- 인당 2~3턴을 기준으로 상황에 맞게 유동적으로 운영
